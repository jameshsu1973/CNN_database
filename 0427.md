## 專案目標

建立一個 CNN 權重資料庫，讓不同任務的模型可以互相分享學習到的權重，實現**遷移學習**。

### 核心功能
- 儲存訓練好的 CNN 權重到向量資料庫
- 根據層的拓撲結構（input/output channels, kernel size）進行相似度搜尋
- 新任務可以從資料庫中取得相似層的權重作為初始化

## 問題與限制

### 1. Zilliz Cloud 免費版限制

| 限制項目 | Zilliz 免費版 | 影響 |
|----------|---------------|------|
| 維度 | ≤ 32,768 | 大型卷積層無法儲存 |
| Collections | ≤ 5 個 | 6 層模型無法全部儲存 |
| 儲存空間 | 5 GB | 暫時足夠 |

### 2. 超過維度限制的層

```
模型層維度分析：
conv_0: 3→32 (3×3)     →   8,640    ✅ 可儲存
conv_1: 32→64 (3×3)   →  73,728    ✅ 可儲存
conv_2: 64→128 (3×3)  → 294,912    ❌ 超過限制
conv_3: 128→256 (3×3) → 737,280    ❌ 超過限制
conv_4: 256→512 (3×3)  → 1,179,648  ❌ 超過限制
conv_5: 512→5 (1×1)   →   2,560    ✅ 可儲存
```

### 3. 嘗試的解決方案

| 方案 | 說明 | 結果 |
|------|------|------|
| 跳過大層 | 超過維度的層不儲存 | ❌ 用戶不希望跳過 |
| 指紋壓縮 | 用 Random Projection 壓縮至 16384 維 | ⚠️ 有損，無法精確還原 |
| 稀疏向量 | 只存非零值 + 索引 | ❌ Zilliz 不支援取出 |
| 遷移 Qdrant | 使用 Qdrant Cloud | 🔄 規劃中 |

## 資料庫架構

### Zilliz Cloud Schema

每個 layer 配置一個 collection，命名格式：
```
cnn_weights_{in_channels}_{out_channels}_{kernel_h}_{kernel_w}
```

**Example**: `cnn_weights_conv_64_128_3_3`

#### Collection Schema

```python
{
    "fields": [
        {"name": "id", "type": "Int64", "is_primary": True},
        {"name": "vector", "type": "FloatVector", "dim": <dimension>},
        {"name": "layer_name", "type": "VarChar", "max_length": 100},
        {"name": "model_name", "type": "VarChar", "max_length": 100},
        {"name": "epoch", "type": "Int32"},
        {"name": "accuracy", "type": "Float"},
        {"name": "shape", "type": "VarChar", "max_length": 100},
        {"name": "layer_key", "type": "VarChar", "max_length": 100},
        {"name": "object_category", "type": "VarChar", "max_length": 50},
        {"name": "num_objects", "type": "Int32"},
        {"name": "use_fingerprint", "type": "VarChar", "max_length": 10},
        {"name": "original_dimension", "type": "Int64"}
    ],
    "auto_id": True
}
```

#### 欄位說明

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | Int64 | 主鍵（自動產生） |
| `vector` | FloatVector | 權重向量或指紋 |
| `layer_name` | VarChar | 層名稱（如 `conv_0`） |
| `model_name` | VarChar | 模型名稱 |
| `epoch` | Int32 | 訓練回合數 |
| `accuracy` | Float | 準確率/mAP |
| `shape` | VarChar | 權重形狀 JSON |
| `layer_key` | VarChar | 層鍵值 |
| `object_category` | VarChar | 物體類別 |
| `num_objects` | Int32 | 訓練樣本數 |
| `use_fingerprint` | VarChar | 是否使用指紋壓縮 |
| `original_dimension` | Int64 | 原始維度（指紋用） |

### 查詢向量生成

#### 方法 1: 拓撲結構查詢（小層）

```python
def _generate_topology_query(self, shape):
    """Generate query vector from layer topology."""
    out_ch, in_ch, kh, kw = shape

    # 編碼為固定維度向量
    # 包含: out_channels, in_channels, kernel_h, kernel_w, 層類型
    base_features = [
        out_ch / 512.0,      # 標準化
        in_ch / 512.0,
        kh / 5.0,
        kw / 5.0,
        1.0 if kh == 3 else 0.5  # 3x3 卷積標記
    ]

    # 擴展到目標維度
    query = np.tile(base_features, target_dim // len(base_features) + 1)
    return query[:target_dim]
```

#### 方法 2: 指紋查詢（大層）

```python
def _compute_fingerprint(weight_tensor, target_dim=16384):
    """Compute fingerprint using Count Sketch."""
    flat = weight_tensor.flatten()
    n_positions = 50  # 每個輸出維度取樣 50 個位置

    indices = np.random.randint(0, len(flat), size=(target_dim, n_positions))
    signs = np.random.choice([-1, 1], size=(target_dim, n_positions))

    fingerprint = np.sum(signs * flat[indices], axis=1) / np.sqrt(n_positions)
    return fingerprint
```

### 使用流程

```python
# 1. 建立 Vault
vault = MilvusWeightVault(
    uri="https://xxx.zilliz.com",
    token="your_token",
    collection_name="cnn_weights"
)

# 2. 訓練模型（冷啟動）
model = create_detection_model(vault, object_category="cat", force_load=False)
# force_load=False → 使用 He 初始化，跳過 vault

# 3. 訓練並儲存
train_model(model)
for module in model.modules():
    if hasattr(module, 'store_weights'):
        module.store_weights(epoch=5, accuracy=45.0)

# 4. 第二輪訓練（使用 vault）
model2 = create_detection_model(vault, object_category="cat", force_load=True)
# force_load=True → 從 vault 載入相似權重
```

### 指紋壓縮邏輯

```
儲存時：
- 維度 > 32768 → 使用指紋壓縮 (16384 維)
- 維度 ≤ 32768 → 直接儲存完整向量

查詢時：
- Collection 維度 = 16384 → 使用指紋查詢
- Collection 維度 < 16384 → 使用拓撲查詢
```

- 維度限制：65,535（2倍於 Zilliz）
- Collections：無限制
- 需要申請 Qdrant Cloud 帳號

- Oracle Cloud Ampere A1 永久免費（4 核, 24GB RAM）
- 完全控制，無任何限制
- 需要設定 VM 和 Docker
